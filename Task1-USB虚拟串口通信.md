## 练习任务

### 1. USB 虚拟串口通信

本任务分为两个核心目标：**下位机主动上报数据** 和 **上位机下发指令控制硬件**，帮助你建立完整的上下位机通信认知。

#### 任务目标一：下位机上报时间戳与运行时长

STM32 通过 USB 虚拟串口持续向上位机发送数据，内容为：

| 通道 | 数据内容 | 说明 |
|---|---|---|
| 通道 1 | 当前时间戳（RTC 或系统 tick 转换） | 格式示例：`2026-05-18 13:30:45` |
| 通道 2 | 上电后运行时长 | 精确到小数点后两位，单位秒。示例：`123.45` |

上位机使用 **VOFA+** 接收并显示这两条数据曲线，第一行（通道 1）显示时间戳，第二行（通道 2）显示运行秒数。

输出的 VOFA+ 数据协议推荐使用 **JustFloat**（高效二进制协议），示例如下：

```
// JustFloat 协议格式：通道0(float) + 通道1(float)
// 伪代码示意
float ch0 = (float)timestamp;   // 时间戳数值
float ch1 = run_seconds;        // 运行秒数，如 123.45
```

#### 任务目标二：上位机下发指令控制板载 LED

上位机通过 VOFA+ 或其他串口工具发送文本命令，STM32 接收后解析并执行相应动作：

| 命令 | 功能 |
|---|---|
| `led_on` | 点亮板载 LED |
| `led_off` | 熄灭板载 LED |
| `led_twinkle`（加分项） | 板载 LED 以 100ms 间隔闪烁，再次发送 `led_off` 停止 |
| `sinx`（加分项） | STM32 模拟生成正弦波数据并通过 VOFA+ 的示波器功能显示波形 |

#### 任务说明

1. 使用 **STM32CubeMX** 配置工程：
   - 开启 USB 外设（Device 模式），选择 CDC（Communication Device Class）虚拟串口。
   - 配置系统时钟（推荐 HSE + PLL，主频根据芯片设定）。
   - 配置一个系统定时器（如 SysTick 或 TIM），用于计时。
   - 配置 UART 对应的 GPIO（用于板载 LED 指示或调试）。
2. 编写代码实现 USB CDC 数据收发：
   - 下位机定时（如每 100ms）向上位机发送 JustFloat 格式数据。
   - 下位机接收上位机下发的字符串命令并解析。
3. 使用 **VOFA+** 验证：
   - 以 JustFloat 协议接收下位机数据，观察时间戳和运行时长曲线。
   - 使用 VOFA+ 的发送面板下发 `led_on` / `led_off` 命令，观察板载 LED 变化。
   - （加分项）发送 `sinx` 命令，在 VOFA+ 示波器面板查看正弦波形。

### 2. 知识点清单

在开始写代码之前，建议先掌握以下知识点：

| 类别 | 知识点 | 在任务中的应用 |
|---|---|---|
| **CubeMX 配置** | 工程创建、时钟树配置、外设使能 | 配置 USB CDC、系统时钟、调试串口 |
| **USB CDC** | USB Device 模式、CDC 协议、HAL 库 USB API | 实现虚拟串口数据收发 |
| **定时器** | SysTick / TIM 配置、定时中断 | 获取上电运行时长、定时上报数据 |
| **串口通信** | 波特率、数据位、停止位 | 理解虚拟串口与物理串口的异同 |
| **数据协议** | JustFloat 格式、浮点数 IEEE 754 编码 | 将数据打包为 VOFA+ 可识别的格式 |
| **VOFA+** | 协议选择、数据绑定、示波器面板 | 接收并可视化下位机数据 |
| **GPIO 控制** | HAL_GPIO_WritePin()、HAL_GPIO_TogglePin() | 控制板载 LED 亮灭 |
| **字符串解析** | `strcmp()`、`strstr()`、`sscanf()` | 解析上位机下发的文本命令 |
| **Git** | clone、add、commit、push | 代码版本管理与提交 |
